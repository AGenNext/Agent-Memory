Memory Model

Built on how memory
actually works

Grounded in Schacter's Seven Sins of Memory, Ebbinghaus forgetting curves, and HSAM research. Perfect memory is pathological. Forgetting is a feature.

effective_confidence = base × e−λ × days
λ is category-specific and user-tunable via config.toml. Facts (λ=0) never decay. Context (λ=0.5) fades in days. Identity (λ=0.001) persists for years.
episodic

Immutable canonical truth

Raw conversation turns. Never superseded. The chat log is more reliable than human recall. epistemic_status = fact, decay_lambda = 0.

identity

Durable person model

Who the human is, how they operate. Decays very slowly (λ=0.001). Still strong after a year without reinforcement.

knowledge

Learned facts

Things shared or discovered. Medium decay (λ=0.02). Reinforced every time it surfaces in a useful recall. Fades without use.

context

Session working context

Current conversation state. Fast decay (λ=0.5). Gone in days. Analogous to working memory — not meant to persist.

instruction

Behavioural directives

"Always answer in bullet points." λ=0. Never decays. Injected into every prompt via the Cortex layer.

uncertainty

Known unknowns

Explicit "we don't know yet" markers emitted by the reconciler. Medium decay — gets resolved or fades.

Architecture

One binary.
Everything inside.

Rust computes. SurrealDB stores and indexes. No network. No process boundary. The entire memory layer runs in-process.

1 AgentMemory (public API) Clean surface: remember, recall, update, forget, resolve_conflict, analytics Rust
2 MemoryService Orchestration, decay computation, reinforcement on recall, reconciler Rust
3 EscalatingRecall 5 tiers: direct → hybrid BM25+vector → superseded → temporal → scope → gap probe Rust
4 CortexSynthesiser Background sleep cycle: intraday synthesis, daily rollup, knowledge brief, active episode Rust
5 ConflictResolver Misinterpretation → version. AgentStandsFirm → show log. FactualContradiction → halt. Rust
6 Store (SurrealDB) BM25 full-text, HNSW vector search, graph traversal, tri-temporal queries, MVCC SurrealDB
7 RocksDB Persistent key-value storage on disk. Embedded, no server, no network. RocksDB
Conflict Resolution

When human and agent
disagree

Three distinct conflict types. Each handled differently. The chat log always outranks human memory. Confidence does not equal accuracy.

— Misinterpretation

"That's not what I meant"

Agent stored an interpretation. Human says it was wrong. Agent accepts, versions the interpretation, logs what was misread. History preserved via supersession chain.

— Agent Stands Firm

"You told me X"

Human claims the agent said something different. Agent retrieves the exact episodic record. Shows verbatim text, timestamp, session ID. Offers a new decision, not a revision.

— Factual Contradiction

"The number was different"

Human disputes a number or verifiable fact from the chat log. Agent shows the exact line. Sets halt_reasoning=true. Does not continue reasoning from the contested value.
