Memory infrastructure for autonomous AI agents

Agent-Memory gives AI agents persistent, searchable, and context-aware memory so they can remember users, projects, decisions, workflows, and outcomes across sessions.

Get Started View Platform-Agent

Agents should not start from zero every time

Most AI agents are stateless. They respond to prompts but forget previous decisions, project history, user preferences, and workflow outcomes. Agent-Memory gives them a dedicated context layer.

Core capabilities

Store, retrieve, summarize, and reuse context across agent workflows.

Persistent Memory

Store important context across sessions so agents can continue work without losing history.

Semantic Search

Retrieve relevant memories using natural-language queries instead of rigid keyword matching.

Project Context

Keep repository, product, customer, or workspace-specific knowledge available to agents.

Session Memory

Track temporary state during active conversations, tasks, and workflows.

Decision Memory

Capture decisions, rationales, and outcomes so future agents understand why something happened.

Runtime Integration

Designed for Platform-Agent, LangGraph, CrewAI, AutoGen, and custom orchestration systems.

How it works

Agent-Memory sits between your agent runtime and the context your agents need to make better decisions.

1. Agent receives a task
2. Requests relevant context
3. Retrieves memories
4. Executes workflow
5. Stores new learnings

Built for real agent products

Use Agent-Memory for AI agent platforms, SaaS copilots, developer tools, product management agents, research agents, customer support agents, workflow automation, and multi-agent applications.

memory.write(
  agent_id="code-review-agent",

  metadata={

    "repo": "AGenNext/Agent-Memory"
  }
)

results = memory.search(
  agent_id="code-review-agent",
  query="What standards apply here?",
  limit=5
)

Example: GitHub Code Review Agent

A code review agent can remember repository architecture, coding standards, past review comments, known technical debt, team preferences, and previous bug patterns.

Pull request opened → Platform-Agent receives webhook → Agent-Memory retrieves project context → Agent reviews the diff → Agent posts comments → Agent-Memory stores the review summary.
