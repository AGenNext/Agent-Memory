Same workflow, different agent stacks

Each implementation uses the same underlying support-agent idea so you can compare framework ergonomics, boilerplate, control flow, typing, and observability.

OpenAI SDK

Raw tool-calling loop for maximum transparency and control over every model and tool interaction.

PydanticAI

Typed tools and structured models for safer inputs, outputs, and validation-heavy workflows.

LangChain

Agent and tool abstractions for teams already using the LangChain ecosystem.

LangGraph

Graph-based state workflows for enforcing multi-step reasoning and review flows.

SurrealDB

Shared knowledge graph and vector-search layer for entity-aware retrieval.

Decision Tracing

Trace retrievals, tool calls, and responses so agent behavior can be inspected over time.

How the frameworks stack up

Quickstart

Clone the repo, install dependencies, load the sample memory layer, and run each agent.

git clone https://github.com/AGenNext/Agent-Memory.git
cd Agent-Memory
uv sync
cp .env.example .env
uv run python load.py
uv run python agent.py
uv run python agent_pydantic.py
uv run python agent_langchain.py
uv run python agent_langgraph.py
