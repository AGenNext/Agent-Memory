What this is

Agent-Memory does not treat memory as a database lookup problem. It treats it as a cognitive process: memories form, strengthen with use, fade without reinforcement, can be reconstructed from temporal anchors, and conflict with each other in ways that require resolution.

Forms & fades

Ebbinghaus decay per category. Every successful recall reinforces a memory and resets its decay clock.

Never overwrites

Supersede-not-overwrite. Updates create a new version; the supersession chain stays queryable. Raw turns are canonical.

Reconstructs

Episodic replay rebuilds an entire past session from a time anchor — not just the fact, but the context around it.

Resolves conflict

Three conflict types, each with a distinct resolution path and a fully logged decision trace.

Quick start

Boots embedded SurrealDB in-process — no server, no install.

use agent_memory::{AgentMemory, MemoryInput, MemoryCategory, RecallQuery};

// Boots embedded SurrealDB — no server, no install
let memory = AgentMemory::open("./data").await?;

// Store with Ebbinghaus decay configured per category
memory.remember(MemoryInput {
    agent_id:   "my-agent".to_string(),
    content:    "User prefers technically precise responses".to_string(),
    category:   MemoryCategory::Identity,
    importance: Some(0.9),
    ..Default::default()
}).await?;

// 5-tier escalating recall with gap protocol
let result = memory.recall_or_gap(query, human_insistence).await?;
# Rust
cargo add agent-memory

# Node.js / TypeScript
npm install @agentnxxt/agent-memory

Memory model

Six categories, each with a default decay rate. Decay is computed at retrieval time, never stored, and every weight is tunable.

Categoryλ (default)Description
episodic0.000Raw conversation turns. Immutable canonical truth. Never decays.
identity0.001Who the person is, how they operate. Very slow decay.
knowledge0.020Learned facts. Decays without reinforcement.
context0.500Current session state. Fades in days.
instruction0.000Behavioural directives. Never decays.
uncertainty0.030Known unknowns emitted by the reconciler.

effective_confidence = base_confidence × e^(−λ × days_since_reinforcement)

Escalating recall

When normal recall finds nothing and the human insists something exists, retrieval escalates through five tiers before emitting a gap probe.

TierStrategy
1Direct lookup — category/scope filter, sub-ms
2Hybrid BM25 + vector (HNSW) merged via Reciprocal Rank Fusion
3Include superseded memories — maybe it was overwritten
4Temporal expansion — 7d / 30d / 90d / 365d windows by known_time
5Scope relaxation — drop session and scope filters
→Gap probe — returns a suggested_prompt for the human

Benchmarks

A reproducible harness drives the public API against synthetic multi-turn fixtures and measures recall@10 plus ingest/recall latency.

cargo run --release --example benchmark
# write a markdown report:
cargo run --release --example benchmark -- --out benchmarks/results/agent-memory.md

Integrity first. Every number the harness reports is measured on the run that produced it. No comparative figures against other frameworks (Mem0, Zep, Letta, LangChain memory) are published until an adapter for each has actually been run on the same fixtures and scoring. See the benchmark methodology for the adapter contract and reproducibility checklist.

Research foundation

Agent-Memory is an Autonomyx original research project, grounded in peer-reviewed cognitive science rather than software-engineering convention.

Reconstruction, not replay

Schacter (Harvard). Confidence ≠ accuracy; source misattribution drives false memory. Basis for the “agent stands firm” conflict type.

Forgetting as a feature

Seven Sins of Memory; HSAM research. Perfect memory is pathological — the agent should not store everything.

Synaptic plasticity

QBI, University of Queensland. Active connections strengthen, unused weaken — reinforcement-on-recall and Ebbinghaus λ.

A map, not a transcript

Build a mental model of where things live. The background synthesiser produces a knowledge brief, not a log dump.
